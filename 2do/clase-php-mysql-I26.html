¿Qué es XAMPP?

XAMPP es un paquete de software gratuito que instala todo lo necesario para tener un servidor web funcionando en tu propia computadora. El nombre es un acrónimo:

X
Cross-platform
Funciona en Windows, Linux y macOS. "Multiplataforma" → andá en cualquier sistema operativo.
A
Apache
El servidor web. Es el programa que recibe las peticiones del navegador y devuelve las páginas PHP procesadas.
M
MariaDB / MySQL
El motor de base de datos. Guarda toda la información en tablas que PHP puede leer y modificar.
P
PHP
El lenguaje del servidor. El código PHP se ejecuta en el servidor antes de enviar el HTML al navegador.
⚠️

¿Por qué necesitamos un servidor? PHP no puede ejecutarse directamente en el navegador como HTML o JavaScript. Necesita un servidor (Apache) que lo procese primero y luego envíe el resultado al navegador.

05

Estructuras de Decisión

Las estructuras de decisión permiten que el programa tome caminos diferentes según una condición.

if

Si la condición es verdadera, ejecuta este bloque.

elseif

Si no, y esta otra condición es verdadera, ejecuta este bloque. Podés tener varios elseif.

else

Si ninguna condición fue verdadera, ejecuta este bloque. Es el "caso por defecto".

PHP — if / elseif / else
<?php $nota = 7; if ($nota >= 9) { echo "Sobresaliente"; } elseif ($nota >= 7) { echo "Muy bien"; } elseif ($nota >= 6) { echo "Aprobado"; } else { echo "Desaprobado"; } // Resultado: "Muy bien" ?>

switch → Cuando hay muchas opciones posibles para una misma variable, es más ordenado que muchos elseif.

PHP — switch
<?php $dia = "lunes"; switch ($dia) { case "lunes": echo "Inicio de semana"; break; case "viernes": echo "¡Casi fin de semana!"; break; case "sábado": case "domingo": // dos casos, mismo resultado echo "Fin de semana"; break; default: // si no coincide ningún case echo "Día de semana"; } ?>
ℹ️

No olvides el break. Sin break, el switch sigue ejecutando todos los casos que siguen aunque no coincidan. Eso se llama "fall-through".

Conexión PHP → MySQL

Para que PHP pueda leer y escribir datos en MySQL necesitamos tres pasos: conectar, operar y cerrar. PHP ofrece dos formas modernas: mysqli (orientado a objetos) y PDO. En clase usamos mysqli porque es más directa para empezar.

⚠️

Datos de conexión en XAMPP por defecto: host = localhost · usuario = root · contraseña = "" (vacía) · puerto = 3306.

7.1

Paso 1 → Crear la base de datos en phpMyAdmin

1️⃣

Abrir phpMyAdmin

Inició XAMPP, asegurate de que Apache y MySQL están en verde. Abrí el navegador y escribís: localhost/phpmyadmin.

2°

Crear la base de datos

Clic en "Nueva" (panel izquierdo) → Escribís el nombre: instituto_i26 → Collation: utf8mb4_unicode_ci → Crear.

3°

Crear la tabla desde SQL

Usá la pestaña SQL y pegá el query de creación de tabla (lo vemos en la sección 09). O usá el asistente visual de phpMyAdmin.

7.2

Paso 2 → Archivo de conexión (conexion.php)

Creamos un archivo separado con los datos de conexión para reutilizarlo en todos los demás archivos con un require_once.

PHP → conexion.php
<?php // ════════════════════════════════════════════════════════════════? // ARCHIVO: conexion.php // Guarda los datos de acceso a la base de datos. // Nunca subas este archivo a GitHub con datos reales. // ════════════════════════════════════════════════════════════════? $host = "localhost"; // Servidor donde corre MySQL (XAMPP = localhost) $usuario = "root"; // Usuario MySQL (XAMPP default) $password = ""; // Contraseña (XAMPP default = vacía) $db = "instituto_i26"; // Nombre de tu base de datos // Crear la conexión $conexion = new mysqli($host, $usuario, $password, $db); // Verificar si la conexión falló if ($conexion->connect_error) { die("Error de conexión: " . $conexion->connect_error); } // Establecer charset (importante para tildes y ñ) $conexion->set_charset("utf8mb4"); // Si llegamos acá sin die(), la conexión fue exitosa ✓ ?>

PHP → cualquier archivo que necesite la DB

CRUD con Formularios

CRUD son las 4 operaciones fundamentales de cualquier sistema con base de datos: Create, Read, Update y Delete. Todo sistema real (Instagram, WhatsApp, tu sistema de alumnos) se reduce a estas cuatro operaciones.

C → CREATE

Insertar un registro nuevo. El usuario llena un formulario → PHP inserta en la tabla con INSERT INTO.

R → READ

Leer y mostrar registros. PHP consulta la tabla con SELECT y muestra los datos en HTML.

U → UPDATE

Modificar un registro existente. El usuario edita datos → PHP actualiza con UPDATE ... SET ... WHERE.

D → DELETE

Eliminar un registro. PHP borra la fila con DELETE FROM ... WHERE.

C

CREATE → Formulario para agregar un alumno

Dos archivos trabajan juntos: el HTML con el form, y el PHP que procesa los datos.

HTML — form_agregar.html
<!-- Formulario de carga --> <form action="agregar.php" method="POST"> <label>Nombre:</label> <input type="text" name="nombre" required> <label>Apellido:</label> <input type="text" name="apellido" required> <label>Email:</label> <input type="email" name="email" required> <button type="submit">Guardar Alumno</button> </form>
PHP — agregar.php (procesa el formulario)
<?php require_once "conexion.php"; // 1. Recibir los datos del formulario (siempre sanitizar!) $nombre = $conexion->real_escape_string($_POST['nombre']); $apellido = $conexion->real_escape_string($_POST['apellido']); $email = $conexion->real_escape_string($_POST['email']); // 2. Construir la query INSERT $sql = "INSERT INTO alumnos (nombre, apellido, email) VALUES ('$nombre', '$apellido', '$email')"; // 3. Ejecutar y verificar resultado if ($conexion->query($sql)) { echo "→ Alumno agregado correctamente."; // Redirigir a la lista después de guardar header("Location: lista.php"); } else { echo "→ Error: " . $conexion->error; } $conexion->close(); ?>

09

MySQL — Lenguaje SQL

SQL (Structured Query Language) es el lenguaje estándar para trabajar con bases de datos relacionales. No es un lenguaje de programación general → es un lenguaje declarativo: le decís qué querés, no cómo obtenerlo.

9.1

Crear una tabla → CREATE TABLE

Una tabla es como una hoja de cálculo: tiene columnas (campos) y filas (registros). Se define con CREATE TABLE.

SQL — CREATE TABLE alumnos
-- Crear la base de datos CREATE DATABASE instituto_i26 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; -- Seleccionar la base de datos activa USE instituto_i26; -- Crear la tabla alumnos CREATE TABLE alumnos ( -- Columna id: entero, auto-incremental, clave primaria id INT NOT NULL AUTO_INCREMENT, -- Cadenas de texto con longitud máxima nombre VARCHAR(100) NOT NULL, apellido VARCHAR(100) NOT NULL, email VARCHAR(200) UNIQUE NOT NULL, -- UNIQUE: dos alumnos no pueden tener el mismo email -- Fecha de inscripción con valor por defecto automático fecha_inscripcion DATE DEFAULT (CURDATE()), -- Boolean en MySQL = TINYINT(1): 1=activo, 0=inactivo activo TINYINT(1) DEFAULT 1, -- Definir la PRIMARY KEY PRIMARY KEY (id) ) ENGINE=InnoDB; -- InnoDB soporta Foreign Keys (MyISAM no)

9.3

Primary Key (PK) → Clave Primaria

⚠️

¿Qué es una Primary Key? Es la columna (o conjunto de columnas) que identifica de forma única e irrepetible cada fila de una tabla. Es como el DNI de un registro. Sin PK, no podés identificar ni editar una fila específica.

1️⃣

Unicidad

No pueden existir dos filas con el mismo valor de PK. Si intentás insertar un duplicado, MySQL devuelve error.

→?

NOT NULL obligatorio

Una PK nunca puede ser NULL. Siempre debe tener un valor. MySQL lo impone automáticamente.

→

Índice automático

MySQL crea un índice interno sobre la PK, haciendo que las búsquedas por id sean extremadamente rápidas.

SQL → Formas de declarar PRIMARY KEY
-- Forma 1: Inline (dentro de la columna) CREATE TABLE productos ( id INT AUTO_INCREMENT PRIMARY KEY, nombre VARCHAR(150) NOT NULL ); -- Forma 2: Al final (más legible, permite PK compuesta) CREATE TABLE inscripciones ( alumno_id INT NOT NULL, materia_id INT NOT NULL, fecha DATE, PRIMARY KEY (alumno_id, materia_id) -- PK compuesta por DOS columnas );
9.4

Foreign Key (FK) → Clave Foránea

⚠️

¿Qué es una Foreign Key? Es una columna en una tabla que hace referencia a la PK de otra tabla. Sirve para relacionar tablas y garantizar que los datos sean coherentes. Es como un vínculo entre registros de distintas tablas.

SQL → Tablas relacionadas con FOREIGN KEY
-- Primero la tabla PADRE (la que no depende de nadie) CREATE TABLE cursos ( id INT AUTO_INCREMENT, nombre VARCHAR(100) NOT NULL, duracion INT DEFAULT 40, -- horas totales PRIMARY KEY (id) ) ENGINE=InnoDB; -- Después la tabla HIJA (la que referencia a la padre) CREATE TABLE alumnos ( id INT AUTO_INCREMENT, nombre VARCHAR(100) NOT NULL, apellido VARCHAR(100) NOT NULL, email VARCHAR(200) UNIQUE, curso_id INT, -- La FK no puede ser NOT NULL si el alumno puede no tener curso PRIMARY KEY (id), -- → Declaración de la FOREIGN KEY CONSTRAINT fk_alumno_curso FOREIGN KEY (curso_id) -- columna en esta tabla REFERENCES cursos(id) -- columna en la tabla padre ON DELETE SET NULL -- si se borra el curso, alumno.curso_id pasa a NULL ON UPDATE CASCADE -- si cambia el id del curso, se actualiza automático ) ENGINE=InnoDB;
9.5

Integridad Referencial

La integridad referencial es la garantía de que los datos entre tablas relacionadas siempre son coherentes y válidos. MySQL la implementa automáticamente mediante las Foreign Keys con el motor InnoDB.

→?

No podés referenciar lo que no existe

Si intentás insertar un alumno con curso_id = 99 y el curso 99 no existe en la tabla cursos, MySQL rechaza la inserción con error. El sistema no permite datos huérfanos.

═══

ON DELETE / ON UPDATE → Reglas de propagación

Definen qué pasa en la tabla hija cuando se modifica o elimina un registro padre:

AcciónEfecto en la tabla hija
RESTRICTImpide borrar el padre si tiene hijos (default)
CASCADEBorra o actualiza los hijos automáticamente
SET NULLPone NULL en la FK de los hijos
NO ACTIONSimilar a RESTRICT, verificación diferida
→?

InnoDB vs MyISAM

Las FK solo funcionan con ENGINE=InnoDB. Con MyISAM, MySQL acepta la sintaxis pero ignora completamente las restricciones. Siempre usá InnoDB en proyectos modernos.

SQL → Ejemplos de integridad referencial en acción
-- → Esto funciona: curso_id=1 existe en la tabla cursos INSERT INTO alumnos (nombre, apellido, curso_id) VALUES ('María', 'Pérez', 1); -- → Esto FALLA: curso_id=999 no existe en la tabla cursos INSERT INTO alumnos (nombre, apellido, curso_id) VALUES ('Carlos', 'López', 999); -- → ERROR 1452 (23000): Cannot add or update a child row: a foreign key constraint fails -- → Esto FALLA si algún alumno tiene ese curso_id (ON DELETE RESTRICT) DELETE FROM cursos WHERE id = 1; -- → ERROR 1451 (23000): Cannot delete or update a parent row: a foreign key constraint fails -- → Con ON DELETE CASCADE, se borran los alumnos del curso automáticamente -- (según cómo definiste la FK)
