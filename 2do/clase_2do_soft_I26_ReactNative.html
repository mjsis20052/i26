¿Qué es React Native?

React Native es un framework de JavaScript creado por Meta (Facebook) que permite crear apps nativas para iOS y Android usando la misma base de código. No es una app web en un browser: genera componentes nativos reales del sistema operativo.

⚛️

Un código, dos plataformas

El mismo archivo .jsx funciona en iOS y Android. Hay diferencias mínimas que se manejan con Platform.

🏎️

Rendimiento nativo

No es un WebView. React Native compila a componentes nativos: View, Text, Image.

♻️

Mismo paradigma que React

Si aprendieron React (3° año), ya saben el 70% de React Native. Hooks, estado, componentes: todo igual.

📦

Expo — la forma fácil

Expo simplifica todo: no necesitan Xcode ni Android Studio para empezar. Un QR y la app corre en el celular.

📱
Diferencia clave: React vs React Native

En React web usamos <div>, <p>, <button>. En React Native usamos <View>, <Text>, <TouchableOpacity>. La lógica y los hooks son idénticos, solo cambian los componentes de UI.

El rol del PHP en nuestra arquitectura

Ya vieron PHP en la clase anterior. En este proyecto, el backend en PHP actúa como el servidor de datos: recibe pedidos de la app móvil (y de la web app de 3°) y responde con JSON. El PHP no sabe ni le importa si quien llama es una app React Native o un browser.

✅
Concepto clave: API agnóstica de cliente

Una buena API REST responde JSON sin importar quién la consume. Eso nos permite que la misma API en PHP sirva tanto a la web app de 3° año (React) como a la app móvil de 2° (React Native).

config/database.php

php 
<?php
// config/database.php — Conexión reutilizable a MySQL

class Database {
    private $host     = 'localhost';
    private $db_name  = 'mi_app_db';
    private $username = 'root';
    private $password = '';
    public  $conn;

    public function getConnection() {
        $this->conn = null;
        try {
            $this->conn = new PDO(
                "mysql:host={$this->host};dbname={$this->db_name}",
                $this->username,
                $this->password
            );
            $this->conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
        } catch (PDOException $e) {
            echo "Error de conexión: " . $e->getMessage();
        }
        return $this->conn;
    }
}

api/usuarios.php — Endpoint REST completo

php · api rest 
<?php
// api/usuarios.php — Responde JSON para GET y POST

// ── CORS: permitir pedidos desde cualquier origen (app móvil)
header("Access-Control-Allow-Origin: *");
header("Content-Type: application/json; charset=UTF-8");
header("Access-Control-Allow-Methods: GET, POST, PUT, DELETE");
header("Access-Control-Allow-Headers: Content-Type, Authorization");

// Incluir conexión
include_once '../config/database.php';
$database = new Database();
$db = $database->getConnection();

$method = $_SERVER['REQUEST_METHOD'];

switch ($method) {

    case 'GET':
        // Devolver todos los usuarios
        $query = "SELECT id, nombre, email, created_at FROM usuarios";
        $stmt  = $db->prepare($query);
        $stmt->execute();
        $usuarios = $stmt->fetchAll(PDO::FETCH_ASSOC);
        echo json_encode(['data' => $usuarios, 'total' => count($usuarios)]);
        break;

    case 'POST':
        // Crear nuevo usuario
        $body  = json_decode(file_get_contents("php://input"));
        $query = "INSERT INTO usuarios (nombre, email) VALUES (:nombre, :email)";
        $stmt  = $db->prepare($query);
        $stmt->bindParam(':nombre', $body->nombre);
        $stmt->bindParam(':email',  $body->email);

        if ($stmt->execute()) {
            http_response_code(201);
            echo json_encode(['mensaje' => 'Usuario creado', 'id' => $db->lastInsertId()]);
        } else {
            http_response_code(500);
            echo json_encode(['error' => 'No se pudo crear el usuario']);
        }
        break;
}

Componente con fetch a la API PHP

jsx · fetch + api php 
// app/usuarios.jsx — Lista de usuarios desde la API
import { useState, useEffect } from 'react';
import { View, Text, FlatList, ActivityIndicator, StyleSheet } from 'react-native';

// ⚠️ Cambiar por la IP de tu computadora en la red local
// No usar "localhost" en el celular: el celular no conoce tu localhost
const API_URL = 'http://192.168.1.50/mi-api/api/usuarios.php';

export default function UsuariosScreen() {
  const [usuarios, setUsuarios] = useState([]);
  const [loading, setLoading]  = useState(true);
  const [error, setError]      = useState(null);

  useEffect(() => {
    fetchUsuarios();
  }, []);

  const fetchUsuarios = async () => {
    try {
      const res  = await fetch(API_URL);
      const json = await res.json();
      setUsuarios(json.data);
    } catch (e) {
      setError('No se pudo conectar al servidor');
    } finally {
      setLoading(false);
    }
  };

  if (loading) return <ActivityIndicator size="large" />;
  if (error)   return <Text style={styles.error}>{error}</Text>;

  return (
    <View style={styles.container}>
      <Text style={styles.titulo}>Usuarios</Text>
      <FlatList
        data={usuarios}
        keyExtractor={(item) => item.id.toString()}
        renderItem={({ item }) => (
          <View style={styles.card}>
            <Text style={styles.nombre}>{item.nombre}</Text>
            <Text style={styles.email}>{item.email}</Text>
          </View>
        )}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 16, backgroundColor: '#f5f2ec' },
  titulo:    { fontSize: 22, fontWeight: 'bold', marginBottom: 16 },
  card:      { backgroundColor: '#fff', padding: 14, borderRadius: 8, marginBottom: 10 },
  nombre:    { fontSize: 16, fontWeight: '600' },
  email:     { fontSize: 13, color: '#7a7060', marginTop: 2 },
  error:     { color: 'red', padding: 20, textAlign: 'center' },
});
⚠️
IP local vs localhost

Cuando la app corre en el celular físico, localhost apunta al celular, no a tu PC. Necesitan usar la IP local de la computadora (ej: 192.168.1.50). Para encontrarla: Windows → ipconfig en cmd · Mac/Linux → ifconfig en terminal. El celular y la PC deben estar en la misma red WiFi.

Misma API, múltiples clientes

Los alumnos de 3° armaron una API REST (Node o PHP) para su web app. La app móvil de 2° va a consumir esa misma API. Los datos que crea un usuario en la web aparecen en el móvil, y viceversa. Eso es integración real.

¿Cómo coordinar con el equipo de 3°?

Para que la integración funcione, ambos equipos necesitan ponerse de acuerdo en el contrato de la API: qué endpoints existen, qué datos devuelven, qué formato tienen.

Ejemplo de integración práctica

Supongamos que 3° hizo un sistema de gestión de tareas. La app de 2° muestra esas mismas tareas en móvil.

jsx · consumir api de 3° 
// services/api.js — Configuración central de la API de 3°

// URL del backend de 3° año (cambiar por la real)
export const API_BASE = 'http://192.168.1.100:3001/api';

export const getTareas = async () => {
  const res = await fetch(`${API_BASE}/tareas`);
  if (!res.ok) throw new Error('Error al obtener tareas');
  return res.json();
};

export const crearTarea = async (titulo) => {
  const res = await fetch(`${API_BASE}/tareas`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ titulo })
  });
  return res.json();
};
